Detailed Description

Motor conversacional completo del asistente robótico.

Orquesta el ciclo completo: Listen -> Detect -> Process -> Speak.

Realiza la invocación de red hacia la interfaz de inferencia del LLMClient.

Se ejecuta con un techo de tiempo duro (self.llm_timeout). Si el cliente LLM se queda colgado por cualquier motivo (DNS, socket, etc. que no respete su propio timeout interno), esto evita que el asistente se quede esperando indefinidamente sin responder nada.

Returns

)
private

Comprueba si la frase contiene alguna pista de fecha/hora.

A diferencia de dateparser.parse() (que exige que casi toda la cadena sea una fecha y por eso nunca detecta nada en frases reales como "tengo cita el martes a las diez"), esta función solo busca pistas léxicas dentro de la frase, sin importar qué más se diga alrededor.

Parameters
textTexto en minúsculas.
Returns
bool True si se detecta alguna referencia temporal.

Procesa una orden verbal para programar un nuevo recordatorio de alerta.

Extrae tanto la FECHA ("el martes", "mañana", "el 5 de julio"...) como la HORA ("a las nueve", "en 20 minutos"...) y combina ambas, igual que se hace para los eventos de calendario. Si solo se da una de las dos, se usa esa; si no se da ninguna, se programa para dentro de 5 minutos.

◆ _loop()

def app.core.assistant_engine.AssistantEngine._loop (   self)
private

Bucle de consumo interno para procesar el texto obtenido del transcriptor de audio.

Intercepta comandos de parada en tiempo real antes de procesar intenciones complejas. Todo el cuerpo va envuelto en try/except: al ser un hilo daemon, una excepción sin capturar lo mata en silencio (sin log visible para el usuario) y el asistente se queda "escuchando" para siempre sin responder nunca más. Esa es la causa más probable de los cuelgues.
