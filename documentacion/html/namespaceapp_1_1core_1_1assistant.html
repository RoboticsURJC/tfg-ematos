◆ get_weather()

def app.core.assistant.get_weather (   city = "Madrid")

Consulta las condiciones meteorológicas en tiempo real de una ciudad.

Realiza un flujo encadenado de peticiones HTTP utilizando las APIs públicas de Open-Meteo:

  1. Geocoding API para transformar el nombre de la ciudad en coordenadas geográficas.
  2. Weather API para obtener la temperatura y velocidad del viento actuales mediante dichas coordenadas.
Parameters
cityNombre textual de la localidad o término de búsqueda a consultar.
Returns
str Cadena resumida lista para el TTS con los grados y viento de la ciudad.
Return values
strMensaje alternativo de error amigable si falla la resolución o conexión de red.

◆ procesar_texto()

str app.core.assistant.procesar_texto ( str  usuario,
str  texto 
)

Orquesta el flujo principal (Pipeline) para procesar un mensaje del usuario.

Ejecuta la toma de decisiones del robot siguiendo cuatro etapas claras:

  1. Invoca el módulo analítico para detectar la intención subyacente del texto.
  2. Bifurca el flujo ejecutando una herramienta local rápida (hora, clima) o delegando al LLM.
  3. Anexa el par de interacción pregunta/respuesta a la estructura de datos en memoria.
  4. Sincroniza y escribe la actualización en el almacenamiento persistente en disco.
Parameters
usuarioNombre o identificador único del usuario que emite el mensaje.
textoTranscripción o entrada de texto limpia del usuario.
Returns
str Respuesta final resuelta que el asistente debe vocalizar o mostrar en pantalla.
